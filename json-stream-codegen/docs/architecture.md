# Architecture and correctness boundaries

## Components

```text
JsonTransformEngine
  ├── QueryParser -> QueryModel.Query
  ├── CanonicalQueryFormatter -> semantic cache key
  ├── JavaSourceGenerator -> direct field-slot expressions
  ├── InMemoryJavaCompiler -> isolated generated class
  ├── SelectiveJsonReader -> Jackson Core token stream + path trie
  ├── JsonExpressionRuntime -> immutable operators and functions
  └── JsonValue -> selected scalar/container values only
```

The parser, AST, formatter, generator, compiler, selective reader, and value model are local to this
library. Jackson Core is the only production dependency; Databind is never loaded at runtime.

## Selective reader

The source generator assigns each distinct field path a stable integer slot. For example:

```sql
SELECT customer.name AS name, amount * 1.1 AS adjusted
WHERE active = true
```

produces slots for `customer.name`, `amount`, and `active`. `SelectiveJsonReader` builds an
immutable trie over those steps. A path step is either an object key (`.name`) or a zero-based
array index (`[0]`), so `items[0].price` and `matrix[1][0]` are ordinary paths. Each trie node
carries two child maps — object children keyed by field name and index children keyed by position —
and a node may legitimately hold both when different queries reach the same name as an object in one
and an array in another. During execution the reader:

1. requires a top-level JSON object;
2. walks every token so malformed unknown content is still rejected;
3. descends selectively when a path prefix matches — object fields against object children, and
   array elements against index children by counting element positions;
4. skips unselected containers and unselected array elements without constructing lists, maps, or
   scalar wrappers;
5. stores selected scalars in a fixed-size array;
6. recursively materializes a container only when that exact path is itself selected;
7. rejects trailing root content.

At runtime only the child map matching the actual JSON shape can fire; keying a non-object or
indexing a non-array simply matches nothing and leaves those slots at JSON null, which is the same
contract as a missing field. An out-of-range array index behaves identically.

The trie precomputes descendant slot arrays. When duplicate object fields occur, a later value
resets the affected slots before replacing them, preserving last-value-wins behavior even when an
entire parent object is replaced by null or a scalar.

If both `profile` and `profile.name` are selected, the reader materializes `profile` once and
derives the descendant from that value. The same holds for arrays: selecting `items` alongside
`items[0].price` materializes `items` once and reads the indexed descendant back out of it rather
than parsing the subtree twice.

## Generated hot path

Generated classes retain only final references to the immutable runtime and reader. A typical body
is structurally:

```java
public String transform(String input) {
    JsonValue[] fields = reader.read(input);
    if (!runtime.isTrue(runtime.equal(fields[2], literal0))) {
        return null;
    }
    return runtime.writeObject2(
            OUTPUT_PREFIXES,
            fields[0],
            runtime.multiply(fields[1], literal1));
}
```

Literal values and already-escaped output-key prefixes are constructed once. One-to-three-field
outputs use fixed-arity serializers, avoiding an output array. Larger projections use one fixed
array and the generic writer.

## JSON representation

`JsonValue` is deliberately smaller than a general JSON tree API:

- null and booleans are shared singletons;
- numbers retain their lexical form and an exact `BigDecimal`;
- selected strings hold decoded Java text;
- selected arrays and objects are immutable copies;
- objects retain encounter order and duplicate keys replace earlier values;
- serialization escapes quotes, backslashes, control characters, and standard JSON escapes.

Generated operations never mutate selected values. Container projection can safely share an
immutable value with the output serializer.

## Validation and resource limits

Query limits:

| Resource | Limit |
|---|---:|
| Query text | 16,384 characters |
| Projection count | 256 |
| Expression-tree depth | 64 |
| Expression AST nodes | 4,096 |

Input limits are explicit Jackson Core `StreamReadConstraints`:

| Resource | Limit |
|---|---:|
| JSON nesting | 1,000 |
| Decoded string | 20,000,000 characters |
| Document | 100,000,000 characters |

These are operational safety limits, not a multi-tenant security sandbox. Generated bytecode runs
with application permissions. Applications accepting unbounded user queries must limit admitted
query cardinality because successful classes and cache entries live for the engine's lifetime.

## Cache and concurrency

The engine uses an exact-text cache and a canonical-query cache. Both map to shared
`CompletableFuture` instances, so equal concurrent compilation requests have one owner. Failed
entries are removed and can be retried. Successful generated transformers are immutable and safe
for concurrent calls; input and output state is method-local.

Each distinct compiled query owns a child classloader. Queries and their classloaders become
collectable when the engine becomes unreachable.

## Compatibility boundary

The query grammar and expression behavior follow a conventional Databind-based transform engine,
while the Java API differs where required to remove Databind:

| Tree engine | This engine |
|---|---|
| `ObjectNode` input | raw JSON `String` input |
| `ObjectNode` output | raw JSON `String` output |
| custom functions use `JsonNode` | custom functions use `JsonValue` |
| complete input tree already exists | only referenced values are materialized |

JSON output is semantically equivalent but not promised to preserve irrelevant whitespace or every
numeric spelling produced by another serializer. Selected input numbers retain their spelling;
computed numbers use `BigDecimal.toString()`.

## Extension rules

- Add a function through `JsonExpressionRuntime.Builder` when existing call syntax is sufficient.
- For new syntax, update `QueryModel`, `QueryParser`, `CanonicalQueryFormatter`, and
  `JavaSourceGenerator` together.
- Add operators to the runtime first, then map them from generated source.
- Preserve full token validation when optimizing skipping; structural substring searches are not a
  correctness-equivalent substitute.
- Benchmark string-in/string-out and allocation, not only expression evaluation over prebuilt
  values.
